# Java 核心语法速览

## 一、Java 与 C++ 的本质区别（先破后立）

| 维度 | C++ | Java | 差异影响 |
|------|-----|------|----------|
| 内存管理 | 手动指针（`new/delete`） | JVM 自动管理（无指针，GC 回收） | 彻底告别内存泄漏/野指针，但失去了==底层控制权== |
| 继承机制 | ==多继承==（`class A : B, C`） | ==单继承 + 多实现==（`class A extends B implements C`） | 结构更清晰，避免菱形继承灾难 |
| 程序入口 | 任意函数 `main` | 必须在类中 `public static void main` | 一切皆对象，纯面向对象 |
| 基础类型 | `int`, `char`, `bool` | 8 种基本类型 + 8 种包装类 | 提供了自动装箱/拆箱，兼顾效率与对象特性 |
| 编译运行 | 编译为机器码 | 编译为字节码（JVM 解释/编译） | 跨平台（`Write Once, Run Anywhere`） |

### 5.new 创建对象流程
<br>

#### (1) 核心结论
构造方法**不创建对象**，仅负责初始化；`new` 是创建对象的入口。

#### (2) 7步执行流程
以 `Student stu = new Student();` 为例：
+ **加载字节码**：加载 `Student.class` 到方法区
+ **栈中声明变量**：在栈内存声明 `stu` 变量，初始为 `null`
+ **堆中开辟空间**：在堆内存为对象分配空间
+ **默认初始化**：成员变量赋系统默认值（如 `0`、`null`）
+ **显示初始化**：执行成员变量的显式赋值
+ **构造方法初始化**：执行构造方法，完成业务初始化
+ **赋地址值**：将堆中对象地址赋值给 `stu`，`stu` 指向该对象

### (一). 类与对象（Class & Object）

```
// 定义类
class Person {
    // 成员变量 (属性)
    String name;
    int age;
    // 构造方法 (创建对象时初始化，无返回值)

// 使用类
public class OOPDemo {
    public static void main(String[] args) {
        // 创建对象 (new 关键字，无需 delete)
        Person p = new Person("张三", 20);
        p.introduce(); // 调用方法
    }
}
```

``` 
    }

```

#### 2. ==继承== (Inheritance)
 
##### (1)***核心思想：代码复用， extends  关键字。***
 
 ```java 
// 父类
class Animal {
    public void eat() {
        System.out.println("动物在吃东西");
    }
}

// 子类继承父类
class Dog extends Animal {
    // 重写方法 (Override)
    @Override
    public void eat() {
        System.out.println("狗在吃骨头");
    }
}
```

---

##### (2)**操作**
将多个子类中重复的代码抽取到父类，子类可以直接使用父类中的这段代码。

---

##### (3)**好处**
- 减少代码冗余，避免重复编写相同逻辑
- 提高代码复用性，让代码更简洁易维护

---

##### (4)**子类真正能继承父类的是什么？**

- **构造方法**：不能被子类继承，可以利用 `super` 关键字调用
- **成员变量**：可以被子类继承，`private` 私有的也可以，但是私有的无法直接调用
- **成员方法**：
  - 虚方法可以被继承
  - `final` 修饰的最终方法不能被继承，可以被调用
  - `static` 修饰的静态方法不能被继承，可以被调用
  - `private` 修饰的私有方法不能被继承，不能被调用
 
---

##### (5) **继承的特点**
- **继承类型规则**：Java 只支持**单继承**，不支持**多继承**，但支持**多层继承**
  - *单继承*：一个子类只能继承一个父类
  - *不能多继承*：子类无法同时继承多个父类
  - *多层继承*：子类可以继承父类，父类又可继承更上层的父类，形成继承链（如 `C extends B`，`B extends A`）

- **父类分类**：
  - 直接父类：子类直接继承的类
  - 间接父类：子类间接继承的上层类（如“爷爷类”）

- **顶级父类**：
  - Java 中的顶级父类是 `java.lang.Object`
  - 所有类都直接或间接地继承自 `Object` 类

---

##### (6) **继承中的成员变量**

+ **书写规则**:
 **抽取共性**：将重复的成员变量提取到父类，子类保留独有变量。

+ **访问规则**:
 **就近原则**：按「局部位置 → 本类成员 → 父类成员」的顺序逐级往上找。
+ **重名变量处理**:

| 写法        | 含义                 |
| ----------- | -------------------- |
| name        | 遵循就近原则访问      |
| this.name   | 明确访问本类成员变量   |
| super.name  | 明确访问父类成员变量   |

---

##### (7) **继承中的方法重写**

+ **什么是方法重写？**
  - 在继承体系中，子类出现了和父类中**一模一样的方法声明**，我们就称子类的这个方法是重写的方法
  - 重写的方法上需要加 `@Override` 注解，用于校验重写的语法是否正确

+ **什么时候使用方法重写？**
  - 当父类的方法不能满足子类要求的时候，需要进行方法重写

+ **重写方法的基本要求**
  - 子类重写的方法声明跟父类保持一致
  - `private` 私有方法、`static` 静态方法、`final` 最终方法 **不能被重写**

<br>

+ **示例代码**
```java
// 父类
class Animal {
    public void eat() {
        System.out.println("动物会吃东西");
    }

    private void sleep() {
        System.out.println("动物会睡觉");
    }

    public static void breathe() {
        System.out.println("动物会呼吸");
    }

    public final void move() {
        System.out.println("动物会移动");
    }
}

// 子类
class Dog extends Animal {
    // 重写父类的eat方法
    @Override
    public void eat() {
        System.out.println("狗吃骨头");
    }

    // 错误示例：不能重写private方法
    // @Override
    // private void sleep() {
    //     System.out.println("狗睡觉");
    // }

    // 错误示例：不能重写static方法
    // @Override
    // public static void breathe() {
    //     System.out.println("狗呼吸");
    // }

    // 错误示例：不能重写final方法
    // @Override
    // public void move() {
    //     System.out.println("狗跑");
    // }

    public void testCall() {
        // this调用：先找本类，再找父类
        this.eat();
        // super调用：直接找父类
        super.eat();
    }
}

// 测试类
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.eat(); // 输出：狗吃骨头
        dog.testCall(); // 输出：狗吃骨头；动物会吃东西
    }
}
```

##### (8) **继承中的构造方法**

+ **继承中构造方法的核心规则**
  - **父类构造方法不可继承**：父类的构造方法不会被子类继承，只能通过 `super` 关键字在子类构造器中调用。必须手动编写 `super(参数列表)`，不可省略。
  - **默认 `super()` 机制**：如果子类构造方法中没有手动书写 `super(...)`，JVM 会自动在子类构造器第一行添加 `super()`，用于调用父类的无参构造方法。
  - **执行顺序**：创建子类对象时，**先执行父类构造方法**，再执行子类构造方法。
<br>

+ **示例代码**

```java
// 父类
public class Fu {
    String name;
    int age;

    // 父类无参构造
    public Fu() {
        System.out.println("父类无参构造执行了~");
    }

    // 父类带参构造
    public Fu(String name, int age) {
        this.name = name;
        this.age = age;
        System.out.println("父类带参构造执行了~");
    }
}

// 子类
public class Zi extends Fu {
    // 子类无参构造（默认隐含 super()）
    public Zi() {
        // super(); // JVM 自动添加，调用父类无参构造
        System.out.println("子类无参构造执行了~");
    }

    // 子类带参构造（手动调用父类带参构造）
    public Zi(String name, int age) {
        super(name, age); // 必须手动编写，调用父类带参构造
        System.out.println("子类带参构造执行了~");
    }
}

// 测试类
public class Test {
    public static void main(String[] args) {
        // 调用子类无参构造：先执行父类无参构造，再执行子类无参构造
        Zi zi1 = new Zi();
        // 调用子类带参构造：先执行父类带参构造，再执行子类带参构造
        Zi zi2 = new Zi("张三", 20);
    }
}
```

##### (9) **==Java 继承结构设计==**

+ **核心条件**:
  - **存在共性内容**：多个类之间有相同的属性（成员变量）和行为（成员方法）
  - **满足 is-a 关系**：子类是父类中的一种（逻辑上属于父类的子类，如“猫 is a 动物”）

---

+ **普通继承设计思路**:
  - 抽取所有相关类的**共性代码**（属性 + 方法）到父类中
  - 子类通过 `extends` 关键字继承父类
  - 子类在复用父类共性功能的基础上，实现自身特有的功能

---

+ ==**复杂继承设计步骤**==:
  - **利用画图法解决**：通过画图梳理类与类之间的层级关系
  - **分类**：将具有相似特征的类归为一组，明确逻辑边界
  - **抽取共性内容（从下往上画）**：从最底层子类开始，逐层向上抽取重复的属性和方法到上层父类
  - **编写代码（从上往下写）**：按照父类→子类的顺序，从顶层父类开始逐层实现代码
 
#### 3. ==多态== (Polymorphism)
##### (1) **核心思想：同一接口，多种实现**。

---

##### (2) **前提**

 + 父类引用指向子类对象(necessary)
 + 有继承/实现关系(necessary)
 + 有方法重写(optional)

---

##### (3)**多态的好处**

+ **提高代码扩展性**
 方法中使用父类型作为参数，可接收父类对象 + 所有子类对象，无需为每个子类单独编写方法。
+ **实现动态绑定**
 若子类重写了父类方法，通过多态调用时，会自动执行对应子类中重写的方法，做到“同一接口，多种实现”。

---

##### (4)**多态调用成员的特点**

- **变量调用**：编译看左边，运行也看左边
  - 编译：检查引用类型（父类）是否存在该变量
  - 运行：执行引用类型（父类）的变量值，不被子类覆盖
- **方法调用**：编译看左边，运行看右边
  - 编译：检查引用类型（父类）是否存在该方法
  - 运行：执行实际对象类型（子类）的重写方法 
  
---

##### (5)**多态的弊端**

- **弊端**：多态环境下，父类引用无法直接调用子类的独有功能
- **原因**：编译时只检查引用类型（父类），子类独有方法不在父类定义中，编译报错
- **解决方案**：向下转型(强制类型转换)
  ```java
  // 格式：子类类型 变量名 = (子类类型) 父类引用;
  if (父类引用 instanceof 子类类型) {
      子类类型 子类变量 = (子类类型) 父类引用;
      子类变量.子类独有方法();
  }
  ```
   + **解决的问题**
将父类引用转换为真实的子类类型，从而调用子类的独有功能（弥补多态的弊端）。
   + **注意事项**
     - **类型不匹配会报错**：转换类型与对象真实类型不一致时，抛出 `ClassCastException`
     - **必须先判断类型**：使用 `instanceof` 关键字校验后再转换

### (三)抽象类和抽象方法

#### 1.**抽象方法**

##### (1) **定义**
  当父类知道子类需要完成某个功能，但不确定子类怎么实现时，父类将该方法定义为**抽象方法**，具体实现交给子类。

##### (2) **格式**
  ```java
   修饰符 abstract 返回值类型 方法名(参数列表);
   ```

##### (3) **特点**
  - 只有方法声明，没有方法体
  - 必须用  abstract  修饰

---

#### 2.**抽象类**

##### (1) **定义**
  包含抽象方法的类，必须定义为抽象类。

##### (2) **格式**
```
修饰符 abstract class 类名 {
    // 成员变量
    // 构造方法
    // 普通方法
    // 抽象方法
}
```

##### (3) **特点**
  - **抽象类不能实例化**（不能 new 对象）
  - 抽象类可以有：成员变量、构造方法、普通方法、抽象方法
  - 抽象类的构造方法是给子类创建对象时使用的

---

#### 3.**抽象类的使用规则**

+ 子类继承抽象父类
+ 子类 **必须重写** 所有抽象方法，否则子类也必须是抽象类
+ 最终必须有一个子类重写全部抽象方法，才能创建对象

---

#### 4.示例代码
```
// 抽象父类
abstract class Animal {
    private String name;

    public Animal(String name) {
        this.name = name;
    }

    // 抽象方法
    public abstract void eat();
    public abstract void sleep();

    public String getName() {
        return name;
    }
}

// 子类
class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public void eat() {
        System.out.println("狗吃骨头");
    }

    @Override
    public void sleep() {
        System.out.println("狗躺着睡");
    }
}

// 测试
public class Test {
    public static void main(String[] args) {
        Dog dog = new Dog("旺财");
        dog.eat();
        dog.sleep();
    }
}
```

---

#### 5.==**抽象类的意义总结**==

+ **强制规范**：规定子类必须具备哪些功能，统一结构。
+ **提高扩展性**：父类只定义标准，不关心实现，便于扩展不同子类。
+ **代码复用**：把共性内容抽到抽象类，子类只需实现个性化部分。
+ **多态基础**：抽象类可作为父类引用，指向子类对象，便于统一管理。

### (四)接口

#### 1.**基础概念**
- **本质**：独立于继承体系的**行为规则**
- **定义**：`public interface 接口名 {
      // 成员变量（常量）
      // 抽象方法
    }`
- **使用**：类与接口是实现关系，用  implements  关键字：
```
public class 类名 implements 接口名 {
    // 重写接口中所有抽象方法
}
```

---

#### 2.**核心规则**
+ 接口**不能实例化**
+ 实现类必须**重写所有抽象方法**，否则自身为抽象类
+ 一个类可**实现多个接口**，也可继承类+实现多个接口

---

#### 3.**成员特点**
- **成员变量**：
  - 默认 `public static final`，只能是常量
  - 必须在定义时赋值，且不可修改。
- **构造方法**：无构造方法,不能创建对象。
- **成员方法**：
  - 本质是抽象方法，默认 `public abstract`（抽象方法）
  - JDK 版本演进：
    - JDK 7 及以前：只能定义抽象方法。
    - JDK 8 新特性：可定义有方法体的默认方法、静态方法。
    - JDK 9 新特性：可定义私有方法（用于代码复用）。

---

#### 4.**关系总结**
- **类与类**：单继承，可多层继承
- **类与接口**：
  + 实现关系：可以单实现、多实现，也可在继承一个类的同时实现多个接口。
  + 注意：多个接口中若有重复抽象方法，实现类只需**重写一次**。
- **接口与接口**：
  + 可以单继承，也可多继承（一个接口可继承多个接口）。
  + 注意：接口继承多个接口时，会继承所有抽象方法，实现类需**重写全部**。

---

#### 5.示例代码

```
// 1. 定义接口（规则）
interface Swim {
    // 常量：默认 public static final
    int MAX_DEPTH = 100;

    // 抽象方法：默认 public abstract
    void swim();
}

interface Fly {
    void fly();
}

// 2. 父类
class Animal {
    public void eat() {
        System.out.println("动物会吃饭");
    }
}

// 3. 子类：继承类 + 实现多个接口
class Duck extends Animal implements Swim, Fly {
    // 重写Swim接口的抽象方法
    @Override
    public void swim() {
        System.out.println("鸭子会游泳，最深下潜" + MAX_DEPTH + "米");
    }

    // 重写Fly接口的抽象方法
    @Override
    public void fly() {
        System.out.println("鸭子会低空飞行");
    }
}

// 4. 测试
public class InterfaceDemo {
    public static void main(String[] args) {
        Duck duck = new Duck();
        duck.eat();    // 继承自Animal
        duck.swim();   // 实现自Swim接口
        duck.fly();    // 实现自Fly接口
    }
}
```

#### 6.接口在JDK 8和9 的新增特性

##### (1) 默认方法（JDK 8）
- **作用**：接口升级时不破坏现有实现类代码
- **格式**：`public default 返回值类型 方法名(参数列表) {}`
- **注意**：
  - 不是抽象方法，不强制重写；重写时需去掉 `default`
  - `public` 可省略，`default` 不可省略
  - 多接口同名默认方法时，实现类必须重写

---

##### (2) 静态方法（JDK 8）
- **作用**：接口级工具方法
- **格式**：`public static 返回值类型 方法名(参数列表) {}`
- **注意**：
  - 只能通过**接口名**调用，不能用实现类/对象调用
  - `public` 可省略，`static` 不可省略

---

##### (3) 私有方法（JDK 9）
- **作用**：抽取默认/静态方法中的重复代码，实现复用
- **格式**：
  1. 实例私有：
     + 作用：为了抽取默认方法中重复代码而出现的
     + 格式：`private 返回值类型 方法名(参数列表) {}`
  2. 静态私有：
     + 作用：为了抽取静态方法中重复代码而出现的
     + 格式：`private static 返回值类型 方法名(参数列表) {}`
- **注意**：仅接口内部可见，供默认/静态方法调用

---

### 示例代码
```java
// 接口定义
interface MyInterface {
    // 1. 默认方法
    default void defaultMethod() {
        System.out.println("默认方法");
        commonCode(); // 调用私有方法
    }

    // 2. 静态方法
    static void staticMethod() {
        System.out.println("静态方法");
        staticCommonCode(); // 调用静态私有方法
    }

    // 3. 私有方法（JDK9+）
    private void commonCode() {
        System.out.println("公共代码（实例）");
    }

    private static void staticCommonCode() {
        System.out.println("公共代码（静态）");
    }
}

// 实现类
class MyImpl implements MyInterface {}

// 测试
public class Test {
    public static void main(String[] args) {
        MyImpl impl = new MyImpl();
        impl.defaultMethod(); // 调用默认方法
        MyInterface.staticMethod(); // 调用静态方法
    }
}
```

#### 3.**对象创建方式**

##### （1） 直接赋值（最常用、推荐）
**特点**：代码简洁，会复用字符串常量池中的对象，性能更优。
`String s = "abc";`

##### (2) new 关键字 + 构造方法
 
| 构造方法 | 说明 |
| :--- | :--- |
| `public String()` | 创建空字符串，内容为 `""` |
| `public String(String original)` | 根据传入字符串创建新对象 |
| `public String(char[] chs)` | 根据字符数组创建字符串对象 |
| `public String(byte[] chs)` | 根据字节数组（ASCII 码）创建字符串对象 |
 
+ 代码示例
```
// 1. 空参构造
String s1 = new String();

// 2. 传入已有字符串
String s2 = new String("abc");

// 3. 传入字符数组
char[] chs = {'a', 'b', 'c'};
String s3 = new String(chs);

// 4. 传入字节数组（对应 ASCII 码）
byte[] bytes = {97, 98, 99};
String s4 = new String(bytes);
```
 
##### (3) 两种创建方式的核心区别

| 对比项 | 直接赋值 | new 关键字 |
| :--- | :--- | :--- |
| **语法示例** | `String str = "Hello";` | `String str = new String("Hello");` |
| **内存位置** | 字符串常量池 | 堆内存 |
| **对象创建** | 常量池中不存在则创建，存在则复用 | 每次都在堆中创建新对象 |
| **内存效率** | 高（复用对象） | 低（可能创建冗余对象） |
| **适用场景** | 推荐使用，大多数情况下的最佳实践 | 需要强制创建新对象时使用 |

#### 5.字符串替换

##### (1)substring 方法

+ 方法定义

#### 3. **常用方法（增删改查）**
 
```
import java.util.ArrayList;

public class Test {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();

        // 增
        list.add("a");          // 末尾添加
        list.add(0, "b");       // 指定索引添加（范围：0~size）

        // 删
        list.remove("a");       // 按元素删，返回boolean
        String s = list.remove(0); // 按索引删，返回被删元素

        // 改
        list.set(0, "c");       // 修改索引0元素，返回旧值

        // 查
        String e = list.get(0); // 获取索引0元素
        int size = list.size(); // 获取元素个数

        System.out.println(list);
    }
}
```

---

#### 4. **关键注意点**
 
- **索引边界**： get/set/remove(index)  索引必须在  0 ~ size-1 ，否则报错。
- **泛型作用**：限定集合存储类型，避免类型转换异常。
- 自动装箱：JDK5+ 基本类型会自动转包装类（如  add(10)  →  add(Integer.valueOf(10)) ）。

**Java 强制处理受检异常，保证程序健壮性。**

``` 
try {
```

## 六、C++ 转 Java 避坑指南
 
#### 1.  ==  不是用来比较字符串内容的！
- C++： `if (str == "abc")`  通常能比较对。
- Java：`if (str == "abc")`  比较的是对象的内存地址。必须用  str.equals("abc") 。
#### 2. 没有指针运算。
- 不能  `int* p = &a`; p++ ，Java 没有指针概念。
#### 3. 文件名与类名。
- 一个  .java  文件中只能有一个  public class ，且文件名必须和这个类名完全一致（大小写敏感）。
#### 4. 没有析构函数。
- 不需要写  ~Person() ，垃圾回收器 GC 会自动回收不再使用的对象。
#### 5. 没有  goto 。
- Java 移除了  goto  关键字，虽然保留为保留字，但不能使用。
