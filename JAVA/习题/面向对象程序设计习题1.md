## 参考答案

1. **B、D**
- A 错误：Java 是单继承 + 多实现，C++ 支持多继承
- C 错误：Java 的 `boolean` 只能是 `true/false`，不能用 0/1

2. **`const`**

3. **`a=12, b=10, c=12`**
- `b = a++`：先赋值再自增，b=10，a=11
- `c = ++a`：先自增再赋值，a=12，c=12

4. **错误，应改用 `s1.equals(s2)`**
- `==` 比较的是地址，两个 `new` 出来的对象地址不同

5. **B、D**
- A 错误：构造方法不能被子类继承
- C 错误：调用父类构造方法应使用 `super(参数)`，`this()` 是调用本类其他构造方法

6. **输出：**
```
猫吃鱼
动物
```
- **方法调用**：编译看左（Animal），运行看右（Cat），执行 Cat 重写的 `eat()`
- **变量调用**：编译看左，运行也看左，访问的是 Animal 的 `name`

7. **A、C、D**
- B 错误：Java 只支持单继承，一个类只能继承一个抽象类

8. **`public abstract`**

9. **③ → ② → ④ → ⑥ → ⑦ → ① → ⑤**
- 加载字节码 → 声明变量 → 分配空间 → 默认初始化 → 显式初始化 → 构造方法初始化 → 赋值地址

10. **参考代码：**
```java
// 抽象类
abstract class Shape {
public abstract double area();
}

// 圆形
class Circle extends Shape {
private double radius;

public Circle(double radius) {
this.radius = radius;
}

@Override
public double area() {
return Math.PI * radius * radius;
}
}

// 矩形
class Rectangle extends Shape {
private double width;
private double height;

public Rectangle(double width, double height) {
this.width = width;
this.height = height;
}

@Override
public double area() {
return width * height;
}
}

// 测试类
public class ShapeTest {
public static void main(String[] args) {
Shape[] shapes = {
new Circle(3), 
new Rectangle(4, 5)
};

for (Shape s : shapes) {
System.out.println("面积：" + s.area());
}
}
}
```

### ==题后回顾==

1. Java 的 final 对应 C++ 的const
  static 在两种语言中都表示"静态/属于类"
2. 继承中构造方法的特点
  + 构造方法不能被子类继承，只能通过 super 调用
  + 创建子类对象时，先父类构造，后子类构造
  + 若父类没有无参构造，子类必须手动 super(参数) 调用有参构造；父类只有有参构造，子类必须手动写 super(参数)，且放在构造方法第一行
3. 多态调用成员特点
  + 方法调用：编译看左边，运行看右边（执行子类重写的方法）
  + 变量调用：编译看左边，运行也看左边（访问父类的变量）
4. 抽象类 vs 接口
  + 抽象类有构造方法（给子类用的），接口没有构造方法
  + **接口中变量默认 public static final,抽象方法，默认带有 public abstract**
> [!NOTE]
>  继承初始化链:
>  接口（无构造） → 只能定义行为规范，没有初始化需求
>  抽象类（有构造，给子类用） → 封装共性的成员变量，通过构造初始化
>  普通子类 → 通过 super() 调用抽象父类构造，完成完整初始化

5. new 对象创建流程理解要点： 先开辟空间，再默认初始化，再显式赋值，最后构造方法初始化——构造方法是最后一步初始化，不是第一步。
