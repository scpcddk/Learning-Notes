# Floyd（弗洛伊德）算法详解

## 一、算法概述

**Floyd-Warshall算法**（简称Floyd算法）是一种解决**加权图中所有顶点对之间最短路径问题**的算法。它采用**动态规划**思想，通过三重循环逐步求解。

### 基本特性：
- **适用性**：有向图、无向图均可
- **边权**：支持正权、负权（但不能有负权环）
- **时间复杂度**：O(n³)
- **空间复杂度**：O(n²)
- **优点**：代码简单，一次计算得到所有点对最短路径

## 二、算法原理

### 1. 动态规划思想

Floyd算法的核心是**逐步允许经过更多的中转顶点**：

```
定义：dist[i][j][k] 表示从顶点i到顶点j，只允许经过顶点1到k作为中间点的最短路径长度
```

### 2. 状态转移方程

```
dist[i][j][k] = min(dist[i][j][k-1], dist[i][k][k-1] + dist[k][j][k-1])
```

**解释**：
- 第1项：不经过顶点k的最短路径
- 第2项：经过顶点k的最短路径（i→k→j）

### 3. 空间优化（降维）

实际实现时，可以使用二维数组，因为每一轮计算只依赖于上一轮结果：

```cpp
// k表示当前允许经过的最大顶点编号
for k from 1 to n:
    for i from 1 to n:
        for j from 1 to n:
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

## 三、算法步骤

### 步骤1：初始化距离矩阵

```cpp
// 创建n×n的距离矩阵dist
for i from 1 to n:
    for j from 1 to n:
        if i == j:
            dist[i][j] = 0           // 自己到自己的距离为0
        else if 存在边(i,j):
            dist[i][j] = weight(i,j) // 边的权值
        else:
            dist[i][j] = INF         // 无穷大，表示不可达
```

### 步骤2：三重循环更新

```cpp
for k from 1 to n:            // 枚举中间点
    for i from 1 to n:        // 枚举起点
        for j from 1 to n:    // 枚举终点
            if dist[i][k] + dist[k][j] < dist[i][j]:
                dist[i][j] = dist[i][k] + dist[k][j]
```

### 步骤3：输出结果

最终`dist[i][j]`存储的就是顶点i到顶点j的最短路径长度。

## 四、详细示例

### 输入图：

```
顶点：1, 2, 3, 4
边： 1-2:1, 2-3:1, 3-4:1, 4-1:1
```

### 初始化矩阵：

|     | 1   | 2   | 3   | 4   |
|-----|-----|-----|-----|-----|
| 1   | 0   | 1   | INF | 1   |
| 2   | 1   | 0   | 1   | INF |
| 3   | INF | 1   | 0   | 1   |
| 4   | 1   | INF | 1   | 0   |

### 执行过程：

**k=1**（允许经过顶点1）：
- 更新dist[3][2]：INF → 2（3→1→2）
- 更新dist[2][3]：INF → 2（2→1→3）

**k=2**（允许经过顶点1,2）：
- 更新dist[1][3]：INF → 2（1→2→3）
- 更新dist[3][1]：INF → 2（3→2→1）

**k=3**（允许经过顶点1,2,3）：
- 更新dist[1][4]：INF → 2（1→3→4）
- 更新dist[4][1]：INF → 2（4→3→1）
- 更新dist[2][4]：INF → 2（2→3→4）
- 更新dist[4][2]：INF → 2（4→3→2）

**k=4**（允许经过所有顶点）：
- 检查所有路径，无更新

### 最终结果：

|     | 1   | 2   | 3   | 4   |
|-----|-----|-----|-----|-----|
| 1   | 0   | 1   | 2   | 1   |
| 2   | 1   | 0   | 1   | 2   |
| 3   | 2   | 1   | 0   | 1   |
| 4   | 1   | 2   | 1   | 0   |

## 五、重要注意事项

### 1. **循环顺序必须是k在最外层**
```cpp
// 正确
for(k) for(i) for(j)

// 错误！  
for(i) for(j) for(k)
```

**原因**：Floyd是动态规划算法，需要保证计算`dist[i][j]`时，`dist[i][k]`和`dist[k][j]`已经包含了前k-1个中转点的最优解。

### 2. **处理重边**
题目中明确提示存在重边，必须取最小权值：
```cpp
dist[u][v] = min(dist[u][v], w);
```

### 3. **防止整数溢出**
当`dist[i][k]`或`dist[k][j]`为INF时，相加可能导致溢出：
```cpp
// 安全做法
if(dist[i][k] < INF && dist[k][j] < INF) {
    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
}
```

### 4. **负权环检测**
算法完成后可以检查是否存在负权环：
```cpp
bool hasNegativeCycle = false;
for(int i = 1; i <= n; i++) {
    if(dist[i][i] < 0) {
        hasNegativeCycle = true;
        break;
    }
}
```

## 六、时间复杂度分析

- **三重循环**：O(n³)
- 适合n ≤ 500的图
- 对于稠密图（边数接近n²），Floyd比运行n次Dijkstra更简单

## 七、应用场景

1. **小规模图的全源最短路径**
2. **传递闭包计算**（可达性判断）
3. **最小环检测**
4. **图的直径计算**（最长最短路径）
5. **动态图的最短路径更新**

## 八、与其他算法比较

| 算法 | 类型 | 时间复杂度 | 特点 |
|------|------|------------|------|
| Floyd | 全源最短路径 | O(n³) | 简单，支持负权边 |
| Dijkstra | 单源最短路径 | O(n²) 或 O(mlogn) | 不支持负权边 |
| Bellman-Ford | 单源最短路径 | O(nm) | 支持负权边，可检测负环 |
| Johnson | 全源最短路径 | O(nmlogn) | 稀疏图全源最优 |

## 九、常见问题解答

### Q1：Floyd算法能处理负权边吗？
**A**：可以，但不能有负权环。如果有负权环，最短路径长度会无限减小。

### Q2：如何记录具体路径？
**A**：需要维护一个`next[i][j]`数组，记录从i到j的最短路径上的下一个点。

```cpp
// 初始化
if(i == j || dist[i][j] == INF) next[i][j] = -1;
else next[i][j] = j;

// 更新
if(dist[i][k] + dist[k][j] < dist[i][j]) {
    dist[i][j] = dist[i][k] + dist[k][j];
    next[i][j] = next[i][k];  // 路径从i到k再到j
}

// 打印路径
void printPath(int i, int j) {
    if(next[i][j] == -1) return;
    cout << i;
    while(i != j) {
        i = next[i][j];
        cout << " -> " << i;
    }
}
```

### Q3：Floyd算法为什么是动态规划？
**A**：因为它具有：
1. **最优子结构**：最短路径的子路径也是最短路径
2. **重叠子问题**：多个路径可能共享相同的子路径
3. **状态转移方程**：dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

### Q4：什么时候使用Floyd算法？
**A**：
- 需要所有点对之间的最短路径
- 图的规模不大（n ≤ 500）
- 需要简单快速的实现
- 图中存在负权边（但无负环）

## 总结

Floyd算法是图论中最经典的全源最短路径算法，虽然时间复杂度较高，但凭借其简洁的实现和强大的功能，在小规模图或需要频繁查询所有点对距离的场景中仍有广泛应用。掌握Floyd算法的关键是理解其动态规划本质和正确的循环顺序。
