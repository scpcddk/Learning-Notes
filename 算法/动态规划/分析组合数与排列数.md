# 组合数与排列数问题分析

## 1. 问题定义

==**组合数**：不考虑元素的顺序，只关心元素的种类和数量。==
- 例如：用1元和2元纸币凑3元，组合数只有2种：`1+1+1` 和 `1+2`

==**排列数**：考虑元素的顺序，不同的顺序算作不同的方案。==
- 例如：用1元和2元纸币凑3元，排列数有3种：`1+1+1`、`1+2`、`2+1`

---

## 2. 动态规划解法对比

### 组合数：
```cpp
for(int i=1;i<=n;i++)  // 先遍历纸币种类
{
    for(int j=a[i];j<=w;j++)  // 再遍历金额
    {
        dp[j] = (dp[j] + dp[j-a[i]]) % mod;
    }
}
```

### 排列数：
```cpp
for(int i=1;i<=w;i++)  // 先遍历金额
{
    for(int j=1;j<=n;j++)  // 再遍历纸币种类
    {
        if(i >= a[j]) 
        {
            dp[i] = (dp[i] + dp[i-a[j]]) % mod;
        }
    }
}
```

## 3. 为什么遍历顺序不同会产生不同结果？

### 组合数（先物品后容量）：
```
初始化：dp[0] = 1, 其他为0

纸币：[1, 2]，目标：3元

i=1 (面额1):
  j=1: dp[1] = dp[1] + dp[0] = 0+1 = 1  (1)
  j=2: dp[2] = dp[2] + dp[1] = 0+1 = 1  (1+1)
  j=3: dp[3] = dp[3] + dp[2] = 0+1 = 1  (1+1+1)

i=2 (面额2):
  j=2: dp[2] = 1 + dp[0] = 1+1 = 2  (1+1, 2)
  j=3: dp[3] = 1 + dp[1] = 1+1 = 2  (1+1+1, 1+2)

结果：dp[3] = 2 (组合数)
```

### 排列数（先容量后物品）：
```
初始化：dp[0] = 1, 其他为0

纸币：[1, 2]，目标：3元

i=1:
  j=1: dp[1] = dp[1] + dp[0] = 0+1 = 1  (1)
  
i=2:
  j=1: dp[2] = dp[2] + dp[1] = 0+1 = 1  (1+1)
  j=2: dp[2] = 1 + dp[0] = 1+1 = 2      (1+1, 2)
  
i=3:
  j=1: dp[3] = dp[3] + dp[2] = 0+2 = 2  (1+1+1, 2+1)
  j=2: dp[3] = 2 + dp[1] = 2+1 = 3      (1+1+1, 2+1, 1+2)

结果：dp[3] = 3 (排列数)
```

---

## 4. 直观理解

### ==**组合数（先物品）**：==
- 固定了考虑纸币的顺序（先考虑1元，再考虑2元）
- 这样就不会出现 `1+2` 和 `2+1` 都被计算的情况
- 相当于每个组合只有一种排列方式被计算

### ==**排列数（先容量）**：==
- 对于每个金额，考虑最后一张纸币是什么
- `1+2`（最后一张是2）和 `2+1`（最后一张是1）会被分别计算
- 相当于考虑了所有可能的顺序

## 5. 总结表格

| 特性 | 组合数（本题） | 排列数（上一题） |
|------|---------------|----------------|
| 遍历顺序 | 先物品，后容量 | 先容量，后物品 |
| 是否考虑顺序 | 否 | 是 |
| 示例：1+2和2+1 | 算1种 | 算2种 |
| 状态定义 | dp[j]: 凑金额j的组合数 | dp[i]: 凑金额i的排列数 |
| 转移方程 | dp[j] += dp[j-a[i]] | dp[i] += dp[i-a[j]] |
| 适用问题 | 组合问题 | 排列问题 |
