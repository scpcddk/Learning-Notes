# 选择排序

## 一.核心思想

==将序列分为“已排序区域”和“未排序区域”，每轮从“未排序区域”中找到最小（升序）/最大（降序）的元素，与未排序区域的第一个元素交换，逐步扩大已排序区域==

---

## 二.算法原理
1. 初始状态：已排序区域为空，未排序区域是整个序列。
2. 第 i 轮（ i 从0开始）：
  - 在未排序区域（下标 i 到 n-1 ）中，找到最小值对应的**下标**: min_idx 。
  - 将**未排序区域的第一个元素（下标 i ）与最小值（下标 min_idx ）交换**，此时下标 i 及之前的元素成为已排序区域。
3. 重复上述步骤，直到未排序区域只剩1个元素（已自然有序）。

---

## 三.==代码==
```
// 选择排序：升序
void selectionSort(int arr[], int n) 
{
    for (int i = 0; i < n - 1; i++) 
    {
        int minIdx = i;
        // 找最小值下标
        for (int j = i + 1; j < n; j++) 
        {
            if (arr[j] < arr[minIdx]) 
            {
                minIdx = j;
            }
        }
        // 交换
        int temp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = temp;
    }
}
```

---

## 四.算法特性
- **时间复杂度**：无论序列是否有序，均为 O(n²) （每轮必须遍历未排序区域找最小值）。
<br>
- **空间复杂度**： O(1) （原地排序，仅用常数额外空间）。
<br>
- **稳定性**：不稳定（例如序列 [2, 2, 1] ，第一轮会将 1 与第一个 2 交换，导致两个 2 的相对位置改变）。

---

## 五.==适用场景==
1. 数据量很小 的场景（几十～几百个元素）
2. 内存极其紧张，必须原地排序（O(1) 额外空间）
3. 交换操作成本很高，但比较成本低（选择排序交换极少，最多 n-1 次）
4. 对稳定性没要求
5. 简单教学、理解排序思想

---

## 六.=="三傻"比较==
- 冒泡排序是相邻元素逐轮交换，每轮将最大/小值“冒泡”到边界；
- 选择排序是先找最值再交换，每轮只交换1次（冒泡排序每轮可能交换多次）。
- 插入排序是将当前元素插入到已排序部分的正确位置，类似整理扑克牌
- 插入排序对数据顺序敏感，基本有序时极快；选择排序交换次数最少，但比较次数固定。小数据量时插入排序通常更优。
