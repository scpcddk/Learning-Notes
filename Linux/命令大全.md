## **四.磁盘管理**

### **(一).ls**

#### 1.作用：
显示指定工作目录下之内容（列出目前工作目录所含的文件及子目录)。

#### 2.语法：
 ls [-alrtAFR] [name...]

#### 3.==参数==：

**可以N个一起拼**，例：
ls -l -a -h =ls - lah

| 参数 | 说明 |
|------|------|
| `-a` 或 `--all` | **显示所有文件（包括以 `.` 开头的隐藏文件）** |
| `-A` 或 `--almost-all` | 显示除 `.` 和 `..` 外的所有文件（包括隐藏文件） |
| `-l` | **以长格式（详细信息）列出文件（权限、所有者、大小、修改时间等）** |
| `-h` 或 `--human-readable` | 与 `-l` 一起使用时，以人类可读的格式显示文件大小（如 KB、MB） |
| `-t` | 按修改时间排序（最新优先） |
| `-r` 或 `--reverse` | 反向排序（配合 `-t`、`-S` 等使用） |
| `-S` | 按文件大小排序（大文件优先） |
| `-R` 或 `--recursive` | 递归列出子目录内容 |
| `-F` 或 `--classify` | 在文件名后附加标识符（`/` 表示目录，`*` 表示可执行文件） |
| `--color` | 彩色输出（通常默认启用，`--color=auto`） |
| `-i` 或 `--inode` | 显示文件的 inode 编号 |
| `-n` 或 `--numeric-uid-gid` | 以数字形式显示 UID 和 GID（替代用户名和组名） |
| `-d` 或 `--directory` | 仅显示目录本身，而非其内容（常用于配合 `-l`） |
| `-1` | 每行只显示一个文件（默认在终端宽度不足时自动启用） |
| `-m` | 以逗号分隔的列表形式显示文件 |
| `-Q` 或 `--quote-name` | 用引号括住文件名（适用于含空格的文件名） |
| `--group-directories-first` | 先显示目录，后显示文件 |
| `--time-style=<格式>` | 自定义时间显示格式（如 `+%Y-%m-%d`） |

### **(二).cd**

#### 1.作用：
- 改变当前工作目录的命令，切换到指定的路径。
- 若目录名称省略，则变换至使用者的 home 目录 (也就是刚 login 时所在的目录)。
- **~ 也表示为 home 目录** 的意思
- **. 则是表示目前所在的目录**
- **.. 则表示目前目录位置的上一层目录**(可以通过连续多次使用 .. 来切换到更高级的目录)
- 使用 **cd - 可以切换到上次访问的目录**
- 可以使用**环境变量**来指定目标目录，并使用 cd 命令切换到该目录(例：`cd $VAR_NAME`)

#### 2.语法：
cd [dirName]
  - dirName：要切换的目标目录，可以是相对路径或绝对路径。
+ 换到绝对路径：指定完整的目录路径来切换到目标目录。
  - cd /path/to/directory
+ 切换到相对路径：指定相对于当前目录的路径来切换到目标目录。
  - cd relative/path/to/directory

### **(三).pwd**

#### 1.作用：
显示当前用户所在的工作目录的完整路径。

#### 2.语法:
pwd [--help][--version]

#### 3.参数：

| 选项 | 说明 |
| :--- | :--- |
| `-L` | 显示逻辑路径（默认行为），解析符号链接 |
| `-P` | 显示物理路径，不解析符号链接 |
| `--help` | 显示帮助信息 |
| `--version` | 显示版本信息 |

#### 4.pwd 命令的实际应用：
+ 检查环境变量 PWD 的值（当使用 -L 选项时）
+ 或者通过遍历父目录来物理确定路径（当使用 -P 选项时）

#### 5.**实际应用**
+ **符号链接目录中时**：
```
$ ln -s /var/www/html myweb
$ cd myweb
$ pwd   //直接输入命令
/home/username/myweb    # 逻辑路径
$ pwd -P
/var/www/html          # 物理路径
```
+ 脚本中使用:
```
#!/bin/bash
CURRENT_DIR=$(pwd)
echo "当前目录是: $CURRENT_DIR"
```

#### 6.注意：
+ pwd 和 echo $PWD 有什么区别？
  pwd 是一个内置命令，而 $PWD 是一个环境变量。在大多数情况下，它们输出相同，但 pwd 命令可以带选项（如 -P），而 $PWD 总是显示逻辑路径。
+ 为什么需要 -P 选项？
  当你在一个符号链接目录中工作时，有时需要知道实际的物理位置而不是符号链接路径，这时 -P 选项就很有用。
+ pwd 命令会失败吗？
  在正常情况下 pwd 不会失败，但如果当前目录被删除（在另一个终端会话中），pwd 可能会显示错误。

#### 7.实践建议
+ **在脚本中使用 pwd -P**：为了确保脚本在任何情况下都能获取到正确的物理路径
+ **结合其他命令使用**：常与 cd、ls 等命令配合使用
+ **注意符号链接**：当路径中有符号链接时，明确知道自己需要的是逻辑路径还是物理路径

#### 8.逻辑路径与物理路径区别
## 逻辑路径与物理路径的区别

| 维度 | 逻辑路径 | 物理路径 |
| :--- | :--- | :--- |
| **定义** | 通过盘符、挂载点、符号链接等逻辑标识的路径 | 硬件存储设备上的真实区块、扇区、磁道等地址 |
| **是否依赖文件系统** | 是，是文件系统层面的抽象 | 是，是存储介质层面的具体位置 |
| **用户是否直接感知** | 是（如 `C:\Users\123\a.txt`） | 否（如磁盘第 1000～2000 号扇区） |
| **是否可变** | 会变（如更换盘符、软链接、挂载点） | 相对稳定（同一文件物理位置一般不随意变动） |
| **映射关系** | 逻辑路径 → 文件系统 → 物理路径 | 直接对应硬件地址 |
|==核心总结==|方便人机交互，屏蔽硬件细节，支持灵活的组织结构（如分区、移动、软链接）|用于底层存储管理、数据恢复、性能优化（如考虑磁盘寻道时间、扇区布局）|

### **(四).mkdir**

#### 1.作用：
创建目录

#### 2.语法：
mkdir [-p] dirName
(-p 确保目录名称存在，不存在的就建一个。)

#### 3.示例：
+ 在工作目录下，建立一个名为 runoob 的子目录 :
  `mkdir runoob`
+ 在工作目录下的 runoob2 目录中，建立一个名为 test 的子目录。
  - 若 runoob2 目录原本不存在，则建立一个。
  - 注：本例若不加 -p 参数，且原本 runoob2 目录不存在，则产生错误。
  `mkdir -p runoob2/test`
