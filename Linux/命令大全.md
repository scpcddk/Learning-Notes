# Linux命令大全

## **一.文件管理**

### **(一).rm**

+ **注意**：文件通过rm命令删除，则无法恢复

#### 1.作用：
删除一个文件或者目录

#### 2.语法：
rm [options] name...

#### 3.参数：
+ -i 删除前逐一询问确认
+ -f 原档案属性设为唯读，亦直接删除，无需逐一确认
+ -r 将目录及以下之档案逐一删除

#### 4.示例：
+ 删除文件与目录
```
# rm  test.txt 
rm：是否删除文件 "test.txt"? y  
# rm  homework  
rm: 无法删除目录"homework": 是一个目录  
# rm  -r  homework  
rm：是否删除 目录 "homework"? y 
# rm  -r  *     ( * 代表当前目录下所有非隐藏文件与目录)
删除当前目录下文件与目录
```

### **(二).mv**

#### 1.作用：
为文件或目录改名、或将文件或目录移入其它位置

#### 2.语法：

| 命令格式 | 运行结果 |
| :--- | :--- |
| `mv source_file` (文件) `dest_file` (文件) | 将源文件名 `source_file` 改为目标文件名 `dest_file` |
| `mv source_file` (文件) `dest_directory` (目录) | 将文件 `source_file` 移动到目标目录 `dest_directory` 中 |
| `mv source_directory` (目录) `dest_directory` (目录) | - 若目录名 `dest_directory` **已存在**，将 `source_directory` 移动到目录 `dest_directory` 中<br>- 若目录名 `dest_directory` **不存在**，则将 `source_directory` 改名为 `dest_directory` |
| `mv source_directory` (目录) `dest_file` (文件) | **出错** |

+ 注意：需要先执行显示隐藏文件命令，否则，隐藏文件以及隐藏文件夹不会被移动到新目录。（英语点号开头的文件/夹会被作为隐藏文件/夹处理）

#### 3.参数：
+ -b: 当目标文件或目录存在时，在执行覆盖前，会为其创建一个备份。
+ -i: 如果指定移动的源目录或文件与目标的目录或文件同名，则会先询问是否覆盖旧文件，输入 y 表示直接覆盖，输入 n 表示取消该操作。
+ -f: 如果指定移动的源目录或文件与目标的目录或文件同名，不会询问，直接覆盖旧文件。
+ -n: 不要覆盖任何已存在的文件或目录。
+ -u：当源文件比目标文件新或者目标文件不存在时，才执行移动操作。

### **(三).cp**

#### 1.作用：
用于复制文件或目录

#### 2.语法：
cp [options] source dest     `//文件或目录路径`

#### 3.参数：
+ -r 或 -R：递归复制目录及其内容（用于复制目录）
+ -i：交互模式，覆盖前提示用户确认
+ -f：强制复制，覆盖目标文件而不提示
+ -v：显示详细的复制过程（verbose）
+ -p：保留文件的原始属性（如权限、时间戳等）
+ -a：归档模式，等同于 -dpR，保留所有文件属性和递归复制目录
+ -u：仅当源文件比目标文件新时才复制（更新模式）
+ -l：创建硬链接而不是复制文件
+ -s：创建符号链接（软链接）而不是复制文件

### **(四).cat**

#### 1.作用：
连接文件并打印到标准输出设备上，查看和连接文件

#### 2.语法：
cat [选项] [文件]

#### 3.参数：
+ -n：显示行号，会在输出的每一行前加上行号
+ -b：显示行号，但只对非空行进行编号
+ -s：压缩连续的空行，只显示一个空行
+ -E：在每一行的末尾显示 $ 符号
+ -T：将 Tab 字符显示为 ^I
+ -v：显示一些非打印字符

#### 4.使用：
+ **显示文件内容**：cat filename 将指定文件的内容输出到终端上(可多个)
+ **连接文件**：cat file1 file2 > combined_file 将 file1 和 file2 的内容连接，并将结果输出到 combined_file 中
+ **创建文件**：cat > filename，输入文本，按 Ctrl+D 保存并退出
+ **在终端显示文件**：将 cat 与管道（|）结合使用，显示其他命令的输出，例如 ls -l | cat 会将 ls -l 的输出通过 cat 打印到终端上
+ **追加重定向**：cat >> filename，将标准输入追加到文件 filename 的末尾
+ **清空 /etc/test.txt 文档内容**：cat /dev/null > /etc/test.txt

## **四.磁盘管理**

### **(一).ls**

#### 1.作用：
列出目前工作目录所含的文件及子目录

#### 2.语法：
 ls [-alrtAFR] [name...]

#### 3.==参数==：

**可以N个一起拼**，例：
ls -l -a -h =ls - lah

| 参数 | 说明 |
|------|------|
| `-a` 或 `--all` | **显示所有文件（包括以 `.` 开头的隐藏文件）** |
| `-A` 或 `--almost-all` | 显示除 `.` 和 `..` 外的所有文件（包括隐藏文件） |
| `-l` | **以长格式（详细信息）列出文件**(权限、所有者、大小、修改时间等)|
| `-h` 或 `--human-readable` | 与 `-l` 一起使用时，以人类可读的格式显示文件大小（如 KB、MB） |
| `-t` | 按修改时间排序（最新优先） |
| `-r` 或 `--reverse` | 反向排序（配合 `-t`、`-S` 等使用） |
| `-S` | 按文件大小排序（大文件优先） |
| `-R` 或 `--recursive` | 递归列出子目录内容 |
| `-F` 或 `--classify` | 在文件名后附加标识符（`/` 表示目录，`*` 表示可执行文件） |
| `--color` | 彩色输出（通常默认启用，`--color=auto`） |
| `-i` 或 `--inode` | 显示文件的 inode 编号 |
| `-n` 或 `--numeric-uid-gid` | 以数字形式显示 UID 和 GID（替代用户名和组名） |
| `-d` 或 `--directory` | 仅显示目录本身，而非其内容（常用于配合 `-l`） |
| `-1` | 每行只显示一个文件（默认在终端宽度不足时自动启用） |
| `-m` | 以逗号分隔的列表形式显示文件 |
| `-Q` 或 `--quote-name` | 用引号括住文件名（适用于含空格的文件名） |
| `--group-directories-first` | 先显示目录，后显示文件 |
| `--time-style=<格式>` | 自定义时间显示格式（如 `+%Y-%m-%d`） |

### **(二).cd**

#### 1.作用：
- 改变当前工作目录的命令，切换到指定的路径。
<br>
- 若目录名称省略，则变换至使用者的 home 目录 (也就是刚 login 时所在的目录)。
- **~ 表示为 home 目录** 的意思
- **. 表示目前所在的目录**
- **.. 表示目前目录位置的上一层目录**(可以通过连续多次使用来切换到更高级的目录)
- 用 **cd - 可以切换到上次访问的目录**
- 用**环境变量**来指定目标目录，并用cd命令切换到该目录(例：`cd $VAR_NAME`)

#### 2.语法：
cd [dirName]  (dirName：要切换的目标目录,相对路径或绝对路径)

+ 换到绝对路径：指定完整的目录路径来切换到目标目录。
  - cd /path/to/directory
+ 切换到相对路径：指定相对于当前目录的路径来切换到目标目录。
  - cd relative/path/to/directory

### **(三).pwd**

#### 1.作用：
显示当前用户所在的工作目录的完整路径

#### 2.语法:
pwd [--help][--version]

#### 3.参数：

| 选项 | 说明 |
| :--- | :--- |
| `-L` | 显示逻辑路径（默认行为），解析符号链接 |
| `-P` | 显示物理路径，不解析符号链接 |
| `--help` | 显示帮助信息 |
| `--version` | 显示版本信息 |

#### 4.pwd 命令的实际应用：
+ 检查环境变量 PWD 的值（当使用 -L 选项时）
+ 通过遍历父目录来物理确定路径（当使用 -P 选项时）

#### 5.**实际应用**
+ **符号链接目录中时**：
```
$ ln -s /var/www/html myweb
$ cd myweb
$ pwd   //直接输入命令
/home/username/myweb    # 逻辑路径
$ pwd -P
/var/www/html          # 物理路径
```
+ 脚本中使用:
```
#!/bin/bash
CURRENT_DIR=$(pwd)
echo "当前目录是: $CURRENT_DIR"
```

#### 6.注意：
+ `pwd` 和 `echo $PWD`有什么区别? 
  pwd 是内置命令，而 `$PWD` 是环境变量。大多数情况，它们输出相同，但 pwd 命令可以带选项（如 -P），而 $PWD 总是显示逻辑路径。
+ 为什么需要 -P 选项？
  当你在一个符号链接目录中工作时，有时需要知道实际的物理位置而不是符号链接路径，这时 -P 选项就很有用。
+ pwd 命令会失败吗？
  正常情况下不会，但如果当前目录被删除（在另一个终端会话中）,可能会显示错误。

#### 7.实践建议
+ **在脚本中使用 pwd -P**：确保脚本在任何情况下都能获取到正确的物理路径
+ **结合其他命令使用**：常与 cd、ls 等命令配合使用
+ **注意符号链接**：当路径中有符号链接时，明确知道自己需要的是逻辑路径还是物理路径

#### 8.逻辑路径与物理路径区别

| 维度 | 逻辑路径 | 物理路径 |
| :--- | :--- | :--- |
| **定义** | 通过盘符、挂载点、符号链接等逻辑标识的路径 | 硬件存储设备上的真实区块、扇区、磁道等地址 |
| **是否依赖文件系统** | 是，是文件系统层面的抽象 | 是，是存储介质层面的具体位置 |
| **用户是否直接感知** | 是（如 `C:\Users\123\a.txt`） | 否（如磁盘第 1000～2000 号扇区） |
| **是否可变** | 会变（如更换盘符、软链接、挂载点） | 相对稳定（同一文件物理位置一般不随意变动） |
| **映射关系** | 逻辑路径 → 文件系统 → 物理路径 | 直接对应硬件地址 |
|==核心总结==|方便人机交互，屏蔽硬件细节，支持灵活的组织结构（如分区、移动、软链接）|用于底层存储管理、数据恢复、性能优化（如考虑磁盘寻道时间、扇区布局）|

### **(四).mkdir**

#### 1.作用：
创建目录

#### 2.语法：
mkdir [-p] dirName
(-p 确保目录名称存在，不存在的就建一个。)

#### 3.示例：
+ 在工作目录下，建立一个名为 runoob 的子目录 :
  `mkdir runoob`
+ 在工作目录下的 runoob2 目录中，建立一个名为 test 的子目录。
  - `mkdir -p runoob2/test`
  - 若 runoob2 目录原本不存在，则建立一个。
  - 注：本例若不加 -p 参数，且原本 runoob2 目录不存在，则产生错误。
