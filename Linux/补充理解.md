# ==一.粘滞位（Sticky Bit）==

## 1.什么是粘滞位？

粘滞位是一种特殊的目录权限，用字母 t 表示，作用是在**共享目录**中限制删除权限

+ 核心规则：只有文件的所有者、目录的所有者、root 才能删除该文件

## 2.粘滞位的设置


### (1).表现形式

| 权限设置 | 表示 | 含义 |
|----------|------|------|
| 没有粘滞位 | `drwxrwxrwx` | 任何人都能删任何文件 |
| 有粘滞位 | `drwxrwxrwt` | 只能删自己的文件 |

### (2).符号模式
```
chmod +t /project      # 添加粘滞位
chmod -t /project      # 移除粘滞位
```

### (2).数字模式
```
chmod 1777 /project    # 1 表示粘滞位，777 表示权限
chmod 1770 /project    # 也可以和其他权限组合
```
+ 数字模式中，粘滞位是最前面的那一位
+ 0:无粘滞位
+ 1:有粘滞位

## 3.常见误区澄清

| 误区 | 真相 |
|------|------|
| 粘滞位保护文件不被修改 | ❌ 只保护**删除**，不保护**修改**（有写权限就能修改内容） |
| 粘滞位对文件有效 | ❌ 粘滞位**只对目录有效**，对文件无效 |
| 有粘滞位就不能删任何文件 | ❌ 仍然可以删**自己**的文件 |
